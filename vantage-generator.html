Vantage. by Havill & Co. Data Builder

Turn a spreadsheet into the app's data file

Edit a familiar workbook, run it through here, and get a fresh vantage-data.json to drop next to the app. No JSON by hand.
⚠️
Clean sources only. Everything in the workbook must come from sources you own — firsthand broker knowledge, public records, leasing packages, company websites. Never CoStar. This tool just reshapes whatever you put in; the sourcing discipline is yours.
🛑
Heads up — this generator is out of date. It does not yet know about the building photos, floor plans, landlord/leasing details, year-built specs, or time-of-day drive times added later, and its sample data predates the hundreds of Google Places amenities that were pulled in. If you regenerate now, those will be wiped from vantage-data.json. Don't run this on your current live data until it's been updated to carry the new fields — ask Claude to update it first.

Get the workbook

Download a starter workbook already filled in with the current buildings, suites, tenants, and amenities. Edit it in Excel — add buildings, change rents, update the tenant roster. Keep it as your master copy.

Build the data file

Pick your edited workbook. The tool checks every row, then gives you a vantage-data.json to download — but only if everything's clean.

    Categories tab — the amenity-type menu ›
    keyShort lowercase tag, no spaces (e.g. coffee, grocery). Amenity rows reference this. Order here = order shown in the app.
    labelDisplay name, e.g. "Bars & nightlife".
    colorHex color for its pins, sliders & chips, e.g. #1FA08C.
    weightedyes = a decision-driver: gets a priority slider, a 0–10 score, and a portfolio bar. no = a display layer: map pin + filter toggle only. Keep the weighted list short (4–6) so the sliders don't get crowded.
    * required  ·  Pre-filled and rarely edited — change it only to add/retire an amenity type.
    Industries tab — the company-type menu ›
    industryOne company type per row, in the order you want them shown in the Competition view. This list is the master menu — every tenant's industry must match one of these. Add a row to introduce a new type; the app picks it up automatically with its own color.
    * required
    Buildings tab — one row per building ›
    idShort lowercase tag, no spaces (e.g. watergarden). This links tenants, suites & amenities to the building — keep it stable.
    nameDisplay name, e.g. The Water Garden.
    addrStreet address.
    submarketSubmarket / pocket label.
    latLatitude (decimal number).
    lngLongitude (decimal number).
    classA, B+, etc. Optional.
    parkinge.g. 3.0 / 1,000. Optional.
    sizeBuilding size text, e.g. ~320,000. Optional.
    availTotal available SF text. Optional.
    rentAsking-rent range text. Optional.
    readThe neighborhood read paragraph. Optional.
    * required
    Suites tab — one row per available suite ›
    building_idMust match an id on the Buildings tab.
    labelSuite label, e.g. Suite 500 or Bldg 7 · Ste 200.
    sfRentable SF as a plain number (e.g. 11450). Shown as "11,450 RSF".
    floorFloor number. Optional.
    descOne-line description. Optional.
    rentAsking rent for the suite, e.g. $5.50/SF FS. Optional.
    statusDefaults to "available". Optional.
    * required
    Tenants tab — one row per company in a building ›
    building_idMust match an id on the Buildings tab.
    companyCompany name.
    industryMust match one of the values on the Industries tab (case doesn't matter). Want a new type? Add it to the Industries tab first.
    * required
    Amenities tab — places near each building ›
    building_idMust match an id on the Buildings tab.
    namePlace name.
    categoryMust match a key on the Categories tab (case doesn't matter). Want a new type? Add it to Categories first.
    latLatitude (decimal number).
    lngLongitude (decimal number).
    ratingGoogle rating, e.g. 4.5.
    blurbShort descriptor. Optional.
    * required  ·  Comes pre-filled with your real Google Places data.
    Access tab — connectivity stats per building › optional
    building_idMust match an id on the Buildings tab.
    labelRow label, shown verbatim, e.g. Freeways, Airports, Transit.
    valueThe text shown, e.g. "7 min to I-10 · 11 min to I-405". Type whatever you want — nothing is auto-calculated.
    * required  ·  One row per stat (a building can have several). Leave a building out entirely and its access block just won't show. This whole tab is optional.
    When the JSON downloads, drop it into the same folder as the app (next to index.html), replacing the old vantage-data.json, then re-deploy.
    To sanity-check before deploying: download the starter workbook, run it straight back through Step 2, and the result should match your current data.
